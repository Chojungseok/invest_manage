# 투자 OS (Invest OS)

> **수익률을 기록하는 앱이 아니라, 투자 판단을 검증 가능한 데이터로 바꾸는 개인 투자 의사결정 OS**

투자 OS는 보유 자산과 수익률만 관리하는 일반적인 자산관리 도구가 아닙니다.  
투자 전에 세운 **가설·근거·리스크·매도 기준**을 기록하고, 일정 기간이 지난 뒤 실제 결과와 비교하여 **투자 판단의 품질을 개선하는 개인 투자 의사결정 관리 시스템**입니다.

---

## 프로젝트가 해결하려는 문제

개인투자자는 투자 결과는 기억하지만, 당시의 판단 과정은 정확하게 남기지 않는 경우가 많습니다.

- 수익이 발생하면 잘 판단했다고 생각하지만 실제로는 운이 좋았을 수 있습니다.
- 손실이 발생하면 잘못된 판단이라고 생각하지만 당시 기준에서는 합리적인 결정이었을 수 있습니다.
- 투자 당시의 가설과 리스크를 사후에 다르게 기억하는 후견지명 편향이 발생합니다.
- 비슷한 감정과 상황에서 같은 투자 실수를 반복해도 이를 데이터로 확인하기 어렵습니다.
- 투자 메모, 포트폴리오, 관심종목, 복기 결과가 서로 분리되어 관리됩니다.

투자 OS는 이러한 문제를 해결하기 위해 **투자 전 판단과 투자 후 결과를 하나의 기록 흐름으로 연결**합니다.

## 핵심 차별점

| 구분 | 일반 자산관리 앱 | 일반적인 트레이딩 저널 | 투자 OS |
|---|---|---|---|
| 중심 질문 | 무엇을 보유하고 얼마나 벌었는가 | 어떤 매매에서 수익·손실이 발생했는가 | 왜 투자했고 그 판단은 타당했는가 |
| 주요 데이터 | 보유량, 수익률, 배당, 자산배분 | 진입·청산, 전략, 손절, 감정 | 가설, 근거, 리스크, 판단점수, 복기 |
| 평가 대상 | 포트폴리오 결과 | 매매 실행 결과 | 투자 의사결정 과정과 결과 |
| 관리 시점 | 현재 보유 상태 중심 | 거래 이후 분석 중심 | 투자 전부터 1·3·6개월 이후까지 |
| 주요 목적 | 자산 현황 파악 | 매매 성과 개선 | 반복되는 판단 오류 발견과 의사결정 개선 |

투자 OS의 핵심은 **수익률과 판단의 품질을 분리해 평가하는 것**입니다.

| 판단 품질 | 투자 결과 | 해석 |
|---|---|---|
| 높음 | 수익 | 좋은 판단이 좋은 결과로 연결됨 |
| 낮음 | 수익 | 잘못된 판단이었지만 운이 좋았을 가능성 |
| 높음 | 손실 | 결과는 손실이지만 판단 과정은 유효했을 가능성 |
| 낮음 | 손실 | 판단 과정과 결과 모두 개선 필요 |

## 투자 의사결정 관리 흐름

```text
관심종목 등록
    ↓
목표 진입가·매수 조건 설정
    ↓
투자 전 판단점수 평가
    ↓
매수·매도·추가매수 등 투자기록 작성
    ↓
1개월·3개월·6개월 판단 리플레이
    ↓
가설과 실제 결과 비교
    ↓
반복되는 판단 오류와 강점 분석
    ↓
다음 투자 원칙에 반영
```

이 흐름을 통해 개별 거래 기록이 단순 메모로 끝나지 않고, 사용자의 투자 판단 능력을 개선하기 위한 데이터로 축적됩니다.

## 주요 기능

### 1. 홈 대시보드

- 총자산과 목표 자산 달성률 확인
- 이번 달 투자금과 목표 월 배당금 관리
- 자산 추이와 자산 유형별 비중 시각화
- 포트폴리오, 투자기록, 복기 일정 요약

### 2. 포트폴리오

- 보유 종목 등록·수정·삭제
- 계좌, 자산 유형, 시장, 통화 기준 관리
- 평가금액, 평가손익, 수익률, 종목 비중 자동 계산
- 계좌·자산 유형별 필터
- 종목별 최대 비중 초과 경고

### 3. 투자기록

다음과 같은 투자 행동을 기록할 수 있습니다.

- 매수
- 매도
- 추가매수
- 관망
- 리밸런싱
- 배당 재투자

각 기록에는 단순 거래정보뿐 아니라 다음과 같은 판단정보를 함께 저장합니다.

- 투자 이유
- 투자 가설
- 예상 리스크
- 감정 상태
- 재검토 기준
- 매도 기준

### 4. 판단점수

투자 전 판단의 질을 7개 평가항목, 총 100점 기준으로 점수화합니다.

판단점수는 향후 수익률과 별도로 관리되며, 사용자가 충분한 검토 없이 감정적으로 투자하고 있는지를 사전에 확인하는 데 활용됩니다.

### 5. 판단 리플레이

매수 기록을 저장하면 1개월·3개월·6개월 복기 일정이 자동으로 생성됩니다.

복기 시 다음 내용을 확인합니다.

- 최초 투자 가설의 현재 유효성
- 복기 시점의 수익률
- 수익 또는 손실의 주요 원인
- 당시 판단에서 잘한 점
- 보완해야 할 판단 요소
- 보유·추가매수·매도·관망 등 다음 행동

### 6. 관심종목

- 목표 진입가 설정
- 매수 조건 기록
- 우선순위 관리
- 현재가의 목표가 도달 여부 확인
- 충동매수를 줄이기 위한 사전 대기목록 운영

### 7. 월간 투자계획

- 월 투자 가능 금액 설정
- 자산군별 목표 비중 설정
- 현재 비중과 목표 비중 비교
- 월별 우선 투자대상 관리
- 목표 비중에 따른 투자금 배분안 확인

### 8. 통계

- 총자산 추이
- 월별 투자금
- 판단점수 추이
- 감정 상태별 투자 결과
- 판단 리플레이 완료율
- 수익 투자와 손실 투자의 평균 판단점수 비교

데이터가 충분히 축적되면 다음과 같은 개인 투자패턴을 확인하는 기반으로 활용할 수 있습니다.

- 높은 판단점수로 투자한 종목의 실제 성과
- 특정 감정 상태에서 실행한 투자의 평균 결과
- 목표 진입가를 지킨 투자와 충동적으로 실행한 투자의 차이
- 추가매수가 효과적이었던 조건
- 손실 투자에서 반복되는 판단 오류

### 9. 설정 및 데이터 관리

- 목표 자산
- 목표 자산배분
- 종목별 최대 비중
- 환율 기준
- 복기 주기
- JSON 데이터 백업 및 복원
- 전체 데이터 초기화

## 주요 사용자

투자 OS는 다음과 같은 개인투자자를 주요 사용자로 가정합니다.

- 매월 일정 금액을 장기적으로 투자하는 사용자
- ETF, 배당주, 성장주와 일부 고위험 종목을 함께 운용하는 사용자
- 단순 수익률보다 투자 원칙과 판단 과정을 중요하게 생각하는 사용자
- 충동매수와 반복되는 판단 오류를 줄이고 싶은 사용자
- 자신의 투자 데이터를 바탕으로 투자 습관을 개선하고 싶은 사용자

초단기 매매 실행을 분석하는 전문 트레이딩 도구보다는, **중장기 투자자의 판단 기록과 복기**에 초점을 맞추고 있습니다.

## 실행 방법

현재 버전은 별도 서버 설치 없이 실행할 수 있는 브라우저 기반 MVP입니다.

### 방법 1. 파일 직접 실행

1. 저장소를 다운로드합니다.
2. `index.html` 파일을 Chrome 또는 Edge 브라우저에서 엽니다.
3. 로컬 계정을 생성한 뒤 투자 데이터를 입력합니다.

### 방법 2. 로컬 웹서버 실행

```bash
git clone https://github.com/Chojungseok/invest_manage.git
cd invest_manage
python -m http.server 8000
```

브라우저에서 `http://localhost:8000`에 접속합니다.

## 로그인 및 데이터 저장 방식

현재 로그인 기능은 서버 인증이 아니라, 하나의 브라우저에서 사용자별 데이터를 구분하기 위한 **로컬 계정 기능**입니다.

- 계정정보와 투자 데이터는 브라우저 `localStorage`에 저장됩니다.
- 비밀번호는 SHA-256 해시 형태로 로컬에 저장됩니다.
- 같은 브라우저 안에서는 계정별 데이터가 분리됩니다.
- 투자 데이터는 외부 서버로 전송되지 않습니다.
- 설정 메뉴에서 JSON 형식으로 데이터를 백업하고 복원할 수 있습니다.

### 현재 구조의 제약

- 다른 기기 또는 브라우저와 데이터가 동기화되지 않습니다.
- 브라우저 데이터를 삭제하면 백업하지 않은 정보는 복구할 수 없습니다.
- 비밀번호를 분실했을 때 별도의 복구 기능이 없습니다.
- 현재 로그인은 실제 서비스 수준의 보안 인증을 제공하지 않습니다.
- 주가와 환율 등 시세정보는 일부 기능을 제외하면 사용자가 직접 입력해야 합니다.

## 데이터 모델

사용자별 투자 데이터는 `localStorage`에 JSON 형태로 저장됩니다.

| 객체 | 설명 |
|---|---|
| `settings` | 목표 자산, 목표 비중, 최대 비중 한도, 환율, 복기 주기 |
| `assets` | 계좌, 종목, 유형, 시장, 통화, 수량, 평균단가, 현재가, 환율 |
| `records` | 투자 행동, 이유, 가설, 리스크, 감정 상태, 재검토·매도 기준 |
| `scores` | 판단점수 평가항목, 총점, 점수 해석 |
| `replays` | 1·3·6개월 복기 일정, 가설 상태, 수익률, 원인, 다음 행동 |
| `watchlist` | 목표 진입가, 매수 조건, 우선순위, 관심종목 상태 |
| `plans` | 월 투자예산, 목표 비중, 우선 투자대상 |
| `snapshots` | 시점별 총자산 스냅샷 |

## 프로젝트 원칙

1. **결과와 판단을 분리한다.**  
   수익이 발생했다고 항상 좋은 판단은 아니며, 손실이 발생했다고 항상 잘못된 판단은 아닙니다.

2. **투자 당시의 생각을 보존한다.**  
   사후 결과를 알고 난 뒤 투자 이유를 바꾸어 기억하지 않도록 최초 가설과 기준을 기록합니다.

3. **감상이 아니라 데이터로 복기한다.**  
   반복되는 실수와 강점을 기록·점수·통계로 확인합니다.

4. **종목 추천보다 사용자의 판단 개선을 우선한다.**  
   투자 OS의 목적은 정답 종목을 알려주는 것이 아니라, 사용자가 더 일관되고 검증 가능한 판단을 하도록 돕는 것입니다.

## 주의사항

이 프로젝트는 개인의 투자기록과 판단 복기를 지원하기 위한 도구입니다.

- 투자 권유 또는 종목 추천 서비스가 아닙니다.
- 표시되는 점수와 통계는 투자수익을 보장하지 않습니다.
- 모든 투자 결정과 그에 따른 책임은 사용자 본인에게 있습니다.
- 실제 서비스로 운영하기 전 인증, 보안, 데이터 백업 구조를 추가로 검토해야 합니다.

## 한 문장으로 설명하면

> **투자 OS는 사용자가 무엇을 샀는지가 아니라, 왜 샀고 그 판단이 시간이 지난 뒤에도 유효했는지를 기록하고 검증하는 개인 투자 의사결정 관리 시스템입니다.**
